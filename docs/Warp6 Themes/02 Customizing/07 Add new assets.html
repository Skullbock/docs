Add new assets

In this tutorial we show you how you can add custom font, css style and javascript files to your template based on the Warp Framework.

The single configuration file /layouts/template.config.php improves setting up the template environment by doing all initialization in one location. It is located in the folder. It initializes all PHP classes, loads all necessary CSS and JavaScript and also all IE specific files. If you need to load custom CSS and JS files, this is where you do it.

How to load a custom CSS file

The configuration file loads all CSS files needed for the template, with template.css being a CSS file that includes all other CSS files. If you want to add custom css pre in the file /css/custom.css. The file is also loaded in the template.config.php: 

	$this->warp->stylesheets->add('css:custom.css');

If you want to add a complete new css file, create the css file in the folder /css e.g. /css/mycss.css and add the following line to the file /layouts/template.config.php: 

	$this->warp->stylesheets->add('css:mycss.css');

How to load a custom JavaScript file

The following lines of pre load the JavaScript used in the template. 

	// javascripts
	$this->warp->javascripts->add('js:warp.js');
	$this->warp->javascripts->add('js:accordionmenu.js');
	$this->warp->javascripts->add('js:dropdownmenu.js');
	$this->warp->javascripts->add('js:template.js');

If you want to add your own script (let's call it myscript.js), simply copy it to the folder /js of the template and add the following line before the closing curly bracket: 

	$this->warp->javascripts->add('js:myscript.js');

How to add custom fonts

Sometimes you need to add a custom font to your Warp theme, this only takes a few steps.

Choose your font

FontSquirrel has a nice service where you can choose between many nice web fonts and you can even upload a custom font from your pc and transform it to a webfont.

Do not break copyrights of custom fonts, not every font is technically build and/or legally licensed for this.
Choose a font which supports Mac Roman subsetting/character encoding, if you want to use french, spanish or other latin languages.

Adding the font to your Theme/Template

Unpack your files to THEME_NAME/fonts/. Please rename the CSS file to something simple like fontname.css

Open THEME_NAME/layouts/template.config.php and add a new line after line ~77, this must should look like this: 
	$fonts = array(
	'fontname => 'template:fonts/fontname.css',
	'bebas' => 'template:fonts/bebas.css',
	'droidsans' => 'template:fonts/droidsans.css',
	'yanonekaffeesatz' => $http.'://fonts.googleapis.com/css?family=Yanone+Kaffeesatz:regular,light');

Open THEME_NAME/config.xml. Now you can add your font to be available in the Warp administration back-end. Actually 3 different types are available (Body, Header and Menu Font). Just add a new option after the lines ~53, ~63, ~73.

Now go to THEME_NAME/css/font1/ copy for example droidsans.css and edit the code to your choosen font file. Repeat this for THEME_NAME/css/font2/ and THEME_NAME/css/font3/ , too.

Use the same property for the font-family value from the CSS file given by THEME_NAME/fonts/fontname.css

Example:

	font-family: 'DroidSansRegular';
