Override system files

This tutorial shows you how to override system files in Joomla and create custom pages in Wordpress.

Override Joomla system files

All Joomla system files are located in /warp/systems/joomla1.x/layouts.

Lets say that you want to customize the output of the content component. To do that, just copy the folder com_content from /warp/systems/joomla1.x/layouts to {TEMPLATEFOLDER}/layouts and assign your changes. This makes sure that you won't lose any of your changes after an update of your template.

Wordpress custom pages

You can use different layouts for pages by creating files in your theme folder with a "page-" as prefix in the name.

Examples:

  • page-{id}.php
  • page-{slug}.php

Lets say you want a custom layout for a page with the id 80:

Create a file in your theme folder and name it page-80.php with the following content. 

		// get warp
		$warp = Warp::getInstance();

		// load main template file, located in /layouts/template-80.php
		echo $warp['template']->render('template-80');

Then go to {YOURTHEME}/layouts and copy and rename template.php to template-80.php. You can now customize template-80.php as you like.
