Permission Issues

This tutorial shows you how to handle permission issues with your YOOtheme Warp template. The following symptoms could result from these problems:

Template staying blank or not displaying correctly

After installing a template your website stays blank, just because your CSS Style sheets and JavaScripts aren't loaded? This might be caused by an internal server error 403, while the webserver tries to access these files.

This is due to incorrect file/folder permissions for the installed template. To execute the .js.php and .css.php files the webserver needs the permission to do so. Thus the templates folder and the files and folders beneath it, need to be set to CHMOD 755.

You can't save the template configuration

In Joomla we use an extra file to be able to save the template settings from the administration, therefore we place a file here: /administrator/templates/system/warp-ajax.php

Please make sure that this folder is writable and has the right permissions, then the template will automatically copy the file. Also you can do it manually just copy the /templates/THEMENAME/warp/systems/joomla.1.5/config/elements/warp-ajax.php to /administrator/templates/system/warp-ajax.php using your favorite FTP manager.

Make sure the template folder (/templates/THEMENAME) and all included sub folders and files are writable.

In WordPress make sure the theme folder (/wp-content/themes/THEMENAME) and all included sub folders and files are writable.

Permissions are inherited from the parent folder. If you set the specific template folder (e.g. yoo_cloud) and all sub files/folders to the correct permission but the whole Joomla template folder has lower permissions it will not work. Best way is to alter the permissions of the top level folder. Sometimes Joomlas root directory has too low permissions and you wonder why it is not working...
Always avoid 777 permissions. If your webserver has problems with the CHMOD 755, you can also try 775 and lastely 777 in this order. Try to use 777 only if you are having problems during installations. After successfull installation, change them back to 755.

How to solve the permissions problem

So how do you change the permissions for the files and folders? Here are two step-by-step guides.

The first guide explains how to use your favorite FTP client such as the FileZilla FTP Client. When using a FTP client, sometimes the ownership of the files/folders you changed the permissions for, changes to the user currently logged in via ftp. This can cause problems. Therefore there is a second solution, which uses eXtplorer, a Joomla! component.

Using your favorite FTP client such as FileZilla FTP Client.

  1. Download FileZilla FTP Client and install it.
  2. Use FileZilla FTP Client to login to your ftp account.
  3. Right click on the template folder and click File Attributes.
  4. In the Change File Attributes dialog, check the options as shown below or type 755. Make sure the Recurse into subdirectories option is checked. Then hit the OK button.
Tip to avoid some trouble: After applying new permissions some FTP clients require the file view to be refreshed. Otherwise the old permissions will continue to show...

For Joomla: Using eXtplorer component

  1. Download eXtplorer and install it. eXtplorer is a file manager. It allows you to edit, delete, copy, rename, archive and unpack files/directories directly on your server.
  2. In the Joomla administration, go to the eXtplorer component and navigate to the template folder. Check the templates folder.
  3. Right click on the folder and choose Change Permissions
  4. Check the options as shown for the ftp client above. Make sure the Recurse into subdirectories option is checked. Then hit the Save button.
